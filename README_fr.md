# Comment Utiliser GPT Image 2 pour l'E-commerce

> Un recueil de prompts pour créer des images e-commerce avec **GPT Image 2** — photos de fiches produit, essayage virtuel, présentations produit, scènes d'interaction et créations social commerce.

## 🍌 Introduction

Ce dépôt vous montre **comment utiliser GPT Image 2 pour créer des images e-commerce** — photos de fiches produit, images principales multi-pays, visuels de détail A+, essayage virtuel, présentations produit, scènes d'interaction et créations social commerce — à l'échelle de la production.

**Chaque cas est une recette :** une photo produit en entrée, le prompt exact à coller dans GPT Image 2 et de vrais exemples de sortie. Remplacez les `{placeholders}` par votre produit, langue, pays et arguments de vente.

Besoin de traiter des centaines de SKU à la fois ? Allez à [🚀 Génération par Lots via l'API Evolink](#-génération-par-lots-via-lapi-evolink).

Essayez sur Evolink : [Workflow GPT Image 2](https://evolink.ai/gpt-image-2?utm_source=github&utm_medium=readme&utm_campaign=gpt-image-2-for-e-commerce)

Si vous le trouvez utile, donnez-nous une Star ⭐

> [!NOTE]

1. **Choisissez un cas** ci-dessous correspondant à votre objectif e-commerce (image principale, essayage modèle, produit multi-angles, visuels A+, etc.).
2. **Téléchargez la photo de votre produit** et **copiez le prompt** de ce cas — remplacez les `{placeholders}` par votre produit, langue, pays, plateforme et arguments de vente.
3. **Exécutez sur GPT Image 2** — soit via l'[interface du workflow Evolink](https://evolink.ai/gpt-image-2?utm_source=github&utm_medium=readme&utm_campaign=gpt-image-2-for-e-commerce) pour les créations ponctuelles, soit via l'[API Evolink](#-génération-par-lots-via-lapi-evolink) pour les SKU en masse et les déploiements multilingues.

## 🖼️ Prompts d'Images pour Fiches Produit

### Cas 1 : Set d'Images pour Marketplace

**Prompt :**

```
Generate a {platform} listing image set for {country} in {language}. The product is {product_name}, and the selling points are {selling_points}.
```

> [!NOTE]
> **Remplacez {platform}, {country}, {language}, {product_name} et {selling_points} avant utilisation.** Ce modèle convient à Amazon, eBay, AliExpress, TEMU, SHEIN, TikTok Shop, Shopee, Lazada, Mercado Libre, Walmart, Etsy, Rakuten, Coupang, Shopify et marketplaces similaires. L'agent planifiera des actifs de fiche adaptés à la plateforme : images sur fond blanc propre, schémas de tailles, vues multi-angles, scènes lifestyle, images d'arguments et détails matières.

## 🚀 Génération par Lots via l'API Evolink

Pour les exécutions en production sur des centaines de SKU, plusieurs langues ou plusieurs marketplaces, appelez directement l'**API Evolink GPT Image 2** au lieu d'exécuter les prompts un par un.

**Pourquoi le batch via API :**

- Génération en masse d'images de fiches pour un catalogue entier
- Traduction multilingue de tous les actifs de fiche en une seule tâche
- Changement de modèle, remplacement de fond et sortie multi-angles programmatiques
- Callbacks webhook s'intègrent avec votre PIM, ERP ou automatisation marketing

**Prérequis :** [Obtenez une clé API](https://evolink.ai/dashboard/keys).

### Étape 1 : Soumettre une tâche de génération

La génération GPT Image 2 est **asynchrone** — l'appel renvoie un `task_id` que vous interrogez pour obtenir les résultats.

```bash
curl -X POST https://api.evolink.ai/v1/images/generations \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "Generate an Amazon listing image set for the US in English. The product is a solid wood dining chair, and the selling points are sturdy frame, ergonomic backrest, easy assembly.",
    "image_urls": ["https://your-cdn.example.com/products/chair.jpg"],
    "size": "1:1",
    "resolution": "2K",
    "quality": "high",
    "n": 4
  }'
```

| Champ | Requis | Notes |
| :--- | :--- | :--- |
| `model` | oui | Doit être `"gpt-image-2"` |
| `prompt` | oui | Jusqu'à 32 000 caractères — collez le prompt de n'importe quel cas de ce dépôt |
| `image_urls` | optionnel | 1–16 images d'entrée/référence (jpeg/png/webp, ≤50 Mo chacune) pour les tâches image-to-image et édition |
| `size` | optionnel | Ratio d'aspect (`1:1`, `2:3`, `16:9`, …) ou dimensions en pixels ; par défaut `auto` |
| `resolution` | optionnel | `1K`, `2K` ou `4K` quand `size` est un ratio |
| `quality` | optionnel | `low`, `medium`, `high` — affecte le coût ; par défaut `medium` |
| `n` | optionnel | 1–10 images par appel |
| `callback_url` | optionnel | Webhook HTTPS pour recevoir le résultat au lieu du polling |

### Étape 2 : Interroger la tâche

```bash
curl https://api.evolink.ai/v1/tasks/{task_id} \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Lorsque `status` devient `"completed"`, le tableau `results` contient les URLs des images générées. **Les images générées sont valides 24 heures — téléchargez-les rapidement.**

### Optionnel : callback webhook

Passez `"callback_url": "https://your-server.example.com/webhook"` dans la soumission et Evolink fera un POST du résultat à votre endpoint à la fin de la tâche — pas de polling.

**Référence API complète :** [docs.evolink.ai · Génération d'images GPT Image 2](https://docs.evolink.ai/en/api-manual/image-series/gpt-image-2/gpt-image-2-image-generation)
