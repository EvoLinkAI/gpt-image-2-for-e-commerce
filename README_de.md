# Wie man GPT Image 2 für E-Commerce nutzt

> Ein Prompt-Rezeptbuch zur Erstellung von E-Commerce-Bildern mit **GPT Image 2** — Listing-Fotos, Model-Anprobe, Produktpräsentationen, Interaktionsszenen und Social-Commerce-Creatives.

## 🍌 Einführung

Dieses Repository zeigt dir, **wie du GPT Image 2 nutzt, um E-Commerce-Bilder zu erstellen** — Listing-Fotos, mehrsprachige Hauptbilder, A+ Detail-Visuals, Model-Anprobe, Produktpräsentationen, Interaktionsszenen und Social-Commerce-Creatives — auf Produktionsniveau.

**Jeder Case unten ist ein Rezept:** ein Produkt-Eingabebild, der exakte Prompt zum Einfügen in GPT Image 2 und echte Ausgabebeispiele. Ersetze die `{placeholders}` durch dein Produkt, deine Sprache, dein Land und deine Verkaufsargumente.

Hunderte SKUs auf einmal verarbeiten? Spring zu [🚀 Batch-Generierung über die Evolink-API](#-batch-generierung-über-die-evolink-api).

Auf Evolink ausprobieren: [GPT-Image-2-Workflow](https://evolink.ai?utm_source=github&utm_medium=readme&utm_campaign=gpt-image-2-for-e-commerce)

Wenn es nützlich ist, gib uns bitte einen Star ⭐

> [!NOTE]
> Dieses Repository konzentriert sich auf wiederverwendbare Prompt-Vorlagen für E-Commerce-Creatives. Ersetze die Platzhalter in geschweiften Klammern vor der Nutzung und passe sie an deine tatsächliche Plattform, Markt, Sprache, Produkt, Material, Modeltyp und Verkaufsargumente an.

## ⚡ Schnellstart

1. **Wähle einen Case** unten, der zu deinem E-Commerce-Ziel passt (Listing-Hauptbild, Model-Anprobe, Multi-Angle-Produkt, A+ Detail-Visuals etc.).
2. **Lade dein Produktfoto hoch** und **kopiere den Prompt** aus diesem Case — ersetze `{placeholders}` durch dein Produkt, Sprache, Land, Plattform und Verkaufsargumente.
3. **Führe ihn auf GPT Image 2 aus** — entweder über die [Evolink-Workflow-UI](https://evolink.ai?utm_source=github&utm_medium=readme&utm_campaign=gpt-image-2-for-e-commerce) für einzelne Creatives oder über die [Evolink-API](#-batch-generierung-über-die-evolink-api) für Massen-SKUs und mehrsprachige Rollouts.

## 🖼️ E-Commerce-Listing-Bild-Prompts

### Case 1: Marketplace-Listing-Bildset

**Prompt:**

```
Generate a {platform} listing image set for {country} in {language}. The product is {product_name}, and the selling points are {selling_points}.
```

> [!NOTE]
> **Ersetze {platform}, {country}, {language}, {product_name} und {selling_points} vor der Nutzung.** Diese Vorlage passt zu Amazon, eBay, AliExpress, TEMU, SHEIN, TikTok Shop, Shopee, Lazada, Mercado Libre, Walmart, Etsy, Rakuten, Coupang, Shopify und ähnlichen Marketplaces. Der Agent plant plattformgerechte Listing-Assets wie saubere Weißhintergrund-Bilder, Größendiagramme, Multi-Angle-Ansichten, Lifestyle-Szenen, Verkaufsargumente-Bilder und Material-Details.

## 🚀 Batch-Generierung über die Evolink-API

Für Produktionsläufe über Hunderte SKUs, mehrere Sprachen oder mehrere Marketplaces rufst du die **Evolink GPT Image 2 API** direkt auf, statt Prompts einzeln auszuführen.

**Warum Batch über API:**

- Massen-Listing-Bildgenerierung für einen ganzen Katalog
- Mehrsprachige Übersetzung aller Listing-Assets in einem einzigen Job
- Programmatischer Modellwechsel, Hintergrund-Austausch, Multi-Angle-Ausgabe
- Webhook-Callbacks integrieren mit deinem PIM, ERP oder Marketing-Automatisierung

**Voraussetzungen:** [API-Key holen](https://evolink.ai/dashboard/keys).

### Schritt 1: Generierungs-Task einreichen

Die GPT-Image-2-Generierung ist **asynchron** — der Aufruf gibt eine `task_id` zurück, die du für Ergebnisse abfragst.

```bash
curl -X POST https://api.evolink.ai/v1/images/generations \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "Generate an Amazon listing image set for the US in English. The product is a solid wood dining chair, and the selling points are sturdy frame, ergonomic backrest, easy assembly.",
    "image_urls": ["https://your-cdn.example.com/products/chair.jpg"],
    "size": "1:1",
    "resolution": "2K",
    "quality": "high",
    "n": 4
  }'
```

| Feld | Pflicht | Hinweise |
| :--- | :--- | :--- |
| `model` | ja | Muss `"gpt-image-2"` sein |
| `prompt` | ja | Bis zu 32.000 Zeichen — füge den Prompt aus jedem Case in diesem Repo ein |
| `image_urls` | optional | 1–16 Eingabe-/Referenzbilder (jpeg/png/webp, je ≤50 MB) für Image-to-Image und Bearbeitung |
| `size` | optional | Seitenverhältnis (`1:1`, `2:3`, `16:9`, …) oder Pixel-Dimensionen; Standard `auto` |
| `resolution` | optional | `1K`, `2K` oder `4K`, wenn `size` ein Verhältnis ist |
| `quality` | optional | `low`, `medium`, `high` — wirkt sich auf die Kosten aus; Standard `medium` |
| `n` | optional | 1–10 Bilder pro Aufruf |
| `callback_url` | optional | HTTPS-Webhook für Fertigstellungs-Empfang statt Polling |

### Schritt 2: Task abfragen

```bash
curl https://api.evolink.ai/v1/tasks/{task_id} \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Wenn `status` `"completed"` wird, enthält das `results`-Array die URLs der generierten Bilder. **Generierte Bilder sind 24 Stunden lang gültig — lade sie zeitnah herunter.**

### Optional: Webhook-Callback

Übergib `"callback_url": "https://your-server.example.com/webhook"` bei der Einreichung, und Evolink sendet das Task-Ergebnis bei Fertigstellung per POST an deinen Endpoint — kein Polling nötig.

**Vollständige API-Referenz:** [docs.evolink.ai · GPT Image 2 Bildgenerierung](https://docs.evolink.ai/en/api-manual/image-series/gpt-image-2/gpt-image-2-image-generation)
